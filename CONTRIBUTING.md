# Contributing to Revoya Documentation

Thank you for your interest in improving Revoya's documentation!

## How to Contribute

### Reporting Issues

Found a typo, broken link, or unclear explanation? Please [open an issue](https://github.com/revoyai/revoya-docs/issues/new) with:

- Page URL or file path
- Description of the problem
- Suggested improvement (if applicable)

### Suggesting Improvements

We welcome suggestions for:

- New documentation pages
- Additional examples
- Clarifications
- Missing information

### Submitting Changes

1. Fork the repository
2. Create a new branch (`git checkout -b improve-docs`)
3. Make your changes
4. Test that all links work
5. Submit a pull request

## Documentation Standards

### Writing Style

- **Clear and concise** - Get to the point quickly
- **Active voice** - "Click the button" not "The button should be clicked"
- **Present tense** - "Revoya scans your code" not "Revoya will scan your code"
- **Second person** - "You can configure..." not "Users can configure..."

### Formatting

- Use sentence case for headings
- Include code examples where helpful
- Add links to related documentation
- Use bullet points for lists
- Use numbered lists for sequential steps

### Code Examples

- Use syntax highlighting
- Include comments for complex examples
- Show both request and response for API examples
- Test all code examples before submitting
